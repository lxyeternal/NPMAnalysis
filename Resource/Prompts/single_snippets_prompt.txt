# Code Context Extraction Prompt

## Task Description
As a code analysis expert, extract the complete context for the flagged code segment. For the flagged line in the Detection information section, provide its surrounding context including control flow and data dependencies.

## CODE TO ANALYZE
```
{CODE}
```

## Context Extraction Rules
1. For the detection point in the Detection information section:
   - Extract the context for that specific line
   - Focus only on this single detection point

2. If the flagged line is within a function/method:
   - Extract the entire function/method definition including decorators/annotations
   - Include any relevant imports or global variables used by the function/method
   - Include any helper functions/methods called by this function/method

3. If the flagged line is outside a function/method (global code):
   - Extract the complete block of global code containing the flagged line
   - Include all control flow structures (if/else, loops, try/catch) that contain or affect the flagged line
   - Include all variable declarations, assignments, and imports that the flagged line depends on
   - Include any function/method or class definitions that the flagged line calls or references
   - If possible, include at least 10 lines before and after the flagged line to provide sufficient context
   - Make sure to capture the full execution path that leads to the flagged line

4. For data dependencies:
   - Trace all variables used in the flagged line to their declaration points
   - Include any functions/methods that manipulate these variables
   - Include constants or configuration values referenced

5. For JavaScript/Node.js specific cases:
   - For require/import statements, include them
   - For npm scripts in package.json, include the entire scripts section
   - For event handlers, include the full event registration and handler
   - For network requests, include the full request configuration and callbacks

## Output Format
Your response must be a valid JSON object with the following fields:

```json
{
  "malicious_code": "// The complete malicious code context with surrounding relevant code",
  "behavior_summary": "Brief description in 20 words or less",
  "evasion_techniques": "Evasion techniques in 20 words or less"
}
```

## IMPORTANT
- You MUST extract ONE context for the single detection point
- Your response must be valid JSON with all three required fields
- Do not include any explanatory text outside the JSON
- Do not include line numbers or detection types in your response
- Analyze both JavaScript and Python code according to their respective language conventions
- Keep behavior_summary and evasion_techniques STRICTLY under 20 words each
- Be concise but precise in your summaries
